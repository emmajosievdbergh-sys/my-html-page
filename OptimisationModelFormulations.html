Final Rolling-Horizon Optimisation Model: Core Formulation

Sets and indices

j ∈ 𝒥  : analysers

h ∈ ℋ  : reagent-linked test families

s ∈ 𝒮  : bottle sizes

k ∈ 𝒦h,s  : bottle identity index

b = (h,s,k) ∈ ℬ  : bottle identity

t ∈ 𝒯  : discrete planning periods

Each bottle identity is tracked individually across time.
Inputs

Fh,t : forecast demand for family h in period t

RKj : rack slot capacity of analyser j

δh,s : initial usable yield of bottle type (h,s)

ℓh : onboard life limit for reagent family h

α : minimum service level

γs : minimum same-day use fraction on opening

M : big-M constant

Initial state: y0,onb, y0,frb, y0,disb, y0,newb, R0b, age0b

Decision and state variables
Location state
yonb,t, yfrb,t, ydisb,t, ynewb,t ∈ {0,1}
Transition events
anewb,t, afrb,t, fb,t, donb,t, dfrb,t ∈ {0,1}
Yield and volume
Rb,t, vb,t, uoldb,t, unewb,t, Lb,t ≥ 0
Demand
zh,t, Uh,t ≥ 0
Age
ageb,t ∈ ℤ≥0
The model tracks bottle location, transitions, remaining yield, served demand, uncovered demand, and bottle age for each period.
Objective function
Stage 1 — minimise uncovered demand
min  ∑h ∑t Uh,t
Fix U* before Stage 2:
∑h ∑t Uh,t ≤ U*
Stage 2 — minimise weighted operational cost
pnew ∑ anewb,t  +  pfr ∑ afrb,t
+ pmv ∑ fb,t  +  pdis ∑ (donb,t + dfrb,t)
+ ploss ∑ Lb,t  +  pon ∑ yonb,t
+ phold ∑ yfrb,t
Stage 2 is solved subject to preserving the Stage 1 service level.
Constraint groups
· State exclusivity
yonb,t + yfrb,t + ydisb,t + ynewb,t = 1  ∀b,t
· Rack capacity
∑{b: home(hb)=j} yonb,t ≤ RKj  ∀j,t
· Transition source constraints
Events can only occur from the correct start-of-day state; at most one load event per bottle per day.
· State balance equations
End-of-day states updated from prior state and transition events: yonb,t = ỹonb,t + anewb,t + afrb,t − fb,t − donb,t, and so on.
· Yield evolution
Rb,t = R̃b,t + δh,s anewb,t − vb,t − Lb,t; consumption and loss bounded by availability.
· Minimum use on opening
unewb,t ≥ γs δh,s anewb,t  ∀b,t
· Age progression and expiry
ageb,t advances by 1 per active day (big-M linearisation); cannot exceed ℓh.
· Demand balance
zh,t + Uh,t = Fh,t  ∀h,t
· Service level constraints
∑h Uh,t ≤ ⌊(1−α) ∑h Fh,t⌋  ∀t  (daily); analogous per-family bound over full window.
· Variable domains
Binary, non-negative, and integer domain restrictions apply.
Single-window optimisation model solved repeatedly in a rolling horizon using updated bottle state and demand forecasts.  ·  ỹ denotes start-of-day value: ỹσb,t = yσb,t−1 for t > 1, equals y0,σb at t = 1.

Yield balance (opening adds δ, consumption and loss reduce):

vb,t = uoldb,t + unewb,t∀ b, t

uoldb,t ≤ R̃b,t∀ b, t
uoldb,t ≤ δh,s(ỹonb,t + afrb,t)∀ b, t
Newly opened volume — upper and lower bounds:
unewb,t ≤ δh,s anewb,t∀ b, t
unewb,t ≥ γs δh,s anewb,t∀ b, t
Loss bounds (only on discard, bounded by available yield):
Lb,t ≤ δh,s(donb,t + dfrb,t)∀ b, t
Lb,t ≤ R̃b,t + δh,s anewb,t∀ b, t
Remaining yield = 0 for inactive bottles:
Rb,t ≤ δh,s(yonb,t + yfrb,t)∀ b, t
Age progression and expiry
Age advances by 1 per active day (big-M linearisation):
ageb,t ≥ age̊b,t + 1 − M(1 − yonb,t − yfrb,t)∀ b, t
ageb,t ≤ age̊b,t + 1 + M(1 − yonb,t − yfrb,t)∀ b, t
Age = 0 when bottle is inactive:
ageb,t ≤ ℓh(yonb,t + yfrb,t)∀ b, t
Hard expiry — cannot exceed onboard life limit:
ageb,t ≤ ℓh∀ b ∈ ℬ, ∀ t ∈ 𝒯
age̊b,t = ageb,t−1 for t > 1; equals age0b at t = 1.

Served demand = total yield consumed from family h:
zh,t = ∑{b: hb=h} vb,t∀ h ∈ ℋ, ∀ t ∈ 𝒯

Per-family service level over full window:
∑t Uh,t ≤ ⌊(1−α) ∑t Fh,t⌋∀ h ∈ ℋ

home(h) assigns each reagent family to exactly one analyser.
Bottles b = (h,s,k) inherit the analyser of their family: home(hb) = home(h).
The constraint is enforced independently per analyser — racks are not shared.
